# PROJECT_IDEA.md — Корпоративный мессенджер "IntraChat"

## 1. Проблема

Сотрудники предприятия общепита (сеть "Вкусно и точка", ~100 чел.) не могут использовать внешние мессенджеры (Telegram, WhatsApp) по регуляторным и корпоративным причинам. При этом директор тратит до 2–3 часов в день на разрозненную коммуникацию через SMS, email и личные встречи. Нет иерархической структуры чатов: сложно разделить операционные отделы, вложенные команды и личные переписки.

## 2. Решение

Корпоративный PWA-мессенджер с акцентом на иерархическую структуру чатов:

1. Администратор создаёт группы верхнего уровня (например, "Ресторан №26219")
2. Внутри группы создаются подгруппы по отделам ("Производство", "Сервис", "Люди")
3. Пользователи организуют личные папки ("Работа", "Семья")
4. Обмен сообщениями: текст, фото/файлы, голос, видео-кружки
5. Статусы доставки/прочтения, реакции, reply, закреп, поиск

## 3. Почему сейчас

- Рост регуляторных ограничений на внешние мессенджеры в корпоративной среде
- PWA-технологии позволяют обойтись без App Store/Play Store — быстрый деплой
- Supabase Realtime делает чат в реальном времени доступным без сложной инфраструктуры

## 4. Целевая аудитория

**Основная:** Директор предприятия + менеджеры отделов (~10 чел.) — управляют коммуникацией
**Вторичная:** Рядовые сотрудники (~90 чел.) — получают задачи, отвечают в чатах

## 5. Архитектура

```
[PWA / Next.js]
    ↓ REST + Realtime (WebSocket)
[Supabase]
    ├── Auth (email+password)
    ├── PostgreSQL (users, groups, messages, folders)
    ├── Storage (файлы, фото, голосовые, видео)
    └── Realtime (live обновление сообщений)
[Vercel] — хостинг фронтенда
[Supabase Cloud] — база данных и хранилище
```

**Стек:** Next.js 14 (App Router), TypeScript, Tailwind CSS, Supabase (Auth + DB + Storage + Realtime), Vercel

## 6. Монетизация

Внутренний продукт, монетизация не требуется. Self-hosted на Supabase Free/Pro план.

## 7. Конкуренты

| Продукт | Что делает | Чего не хватает |
|---|---|---|
| Telegram | Папки, группы, боты | Нельзя использовать корпоративно |
| WhatsApp | Группы | Нет папок, нет контроля |
| Slack | Каналы, треды | Дорого, внешний сервис |
| Teams | Корпоративный чат | Сложно, требует O365 |

## 8. План запуска

**MVP (4–6 недель):**
- Аутентификация, управление пользователями
- Группы и подгруппы (2 уровня)
- Личные папки пользователя
- Текстовые сообщения в реальном времени
- Отправка фото и файлов
- Статус доставки/прочтения
- Reply на сообщение
- Реакции (эмодзи)
- Голосовые сообщения
- Видео-кружки
- Закреп сообщений
- Поиск по чатам
- PWA (добавить на домашний экран)

**v2 (после MVP):**
- Нативное мобильное приложение (React Native)
- Push-уведомления
- Звонки (аудио/видео)
- Боты / интеграции

## 9. Риски

| Риск | Вероятность | Митигация |
|---|---|---|
| Supabase Realtime под нагрузкой 100+ чел. | Средняя | Мониторинг, upgrade плана |
| Хранилище файлов растёт быстро | Высокая | Лимиты на размер файлов, CDN |
| PWA не поддерживает push на iOS | Средняя | iOS 16.4+ поддерживает, документировать |

## 10. Техдетали

**Ключевые таблицы:** users, groups, group_members, messages, message_reactions, message_attachments, folders, folder_chats

**Реальное время:** Supabase Realtime Channels на таблицу messages (INSERT)

**Хранилище:** Supabase Storage bucket "attachments" — фото, файлы, голосовые (.webm), видео-кружки (.mp4)
