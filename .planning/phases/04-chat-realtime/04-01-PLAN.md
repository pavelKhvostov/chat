**src/lib/actions/messages.ts**

Экспортируй тип `MessageWithRelations` (определён в секции interfaces выше).

Константа `PAGE_SIZE = 50`.

Функция `fetchMessages(groupId: string, cursor?: string): Promise<MessageWithRelations[]>`:
- Создать supabase клиент через `await createClient()`
- Запрос к таблице `messages` с select:
  ```
  *,
  sender:profiles!sender_id(id, display_name, avatar_url),
  reply:messages!reply_to(id, content, sender:profiles!sender_id(display_name)),
  reactions:message_reactions(id, emoji, user_id),
  reads:message_reads(user_id)
  ```
- `.eq('group_id', groupId)`
- `.order('created_at', { ascending: false })`
- `.limit(PAGE_SIZE)`
- Если передан `cursor`: добавить `.lt('created_at', cursor)`
- Если error — `throw error`
- Вернуть `(data ?? []).reverse() as MessageWithRelations[]`

Функция `sendMessage(groupId: string, content: string, replyTo?: string): Promise<void>`:
- Получить user через `supabase.auth.getUser()`, если нет — throw new Error('Unauthorized')
- INSERT в `messages`: `{ group_id: groupId, content, sender_id: user.id, reply_to: replyTo ?? null }`
- Если error — throw error

Функция `deleteMessage(messageId: string): Promise<void>`:
- Получить user, если нет — throw
- UPDATE `messages` SET `deleted_at = new Date().toISOString()`
- WHERE `id = messageId` AND `sender_id = user.id` (дополнительная защита помимо RLS)
- Если error — throw

Функция `markMessagesAsRead(messageIds: string[]): Promise<void>`:
- Если `messageIds.length === 0` — return сразу
- Получить user, если нет — throw
- Upsert в `message_reads`:
  ```typescript
  await supabase
    .from('message_reads')
    .upsert(
      messageIds.map((message_id) => ({ message_id, user_id: user.id })),
      { onConflict: 'message_id,user_id', ignoreDuplicates: true }
    )
  ```
- Ошибку логировать в console.error, не бросать (чтобы не ломать UX при частичном сбое)

**src/lib/actions/reactions.ts**

Функция `toggleReaction(messageId: string, emoji: string): Promise<void>`:
- Получить user, если нет — throw
- Проверить существующую реакцию:
  ```typescript
  const { data: existing } = await supabase
    .from('message_reactions')
    .select('id')
    .eq('message_id', messageId)
    .eq('user_id', user.id)
    .eq('emoji', emoji)
    .maybeSingle()
  ```
- Если `existing` — DELETE по `id`
- Если нет — INSERT `{ message_id: messageId, emoji, user_id: user.id }`
- Если error на любой операции — throw

**src/hooks/useRealtime.ts**

Импорты: `useEffect, useRef` из react; `createClient` из `@/lib/supabase/client`; `Tables` из `@/lib/types/database.types`.

Тип `RealtimeMessage = Tables<'messages'>`.

Интерфейс `UseRealtimeOptions`:
```typescript
interface UseRealtimeOptions {
  groupId: string
  onInsert: (msg: RealtimeMessage) => void
  onUpdate: (msg: RealtimeMessage) => void
  onDelete: (id: string) => void
}
```

Функция `useRealtime({ groupId, onInsert, onUpdate, onDelete }: UseRealtimeOptions): void`:
- `const supabase = createClient()` — вне useEffect (один экземпляр)
- `const channelRef = useRef<ReturnType<typeof supabase.channel> | null>(null)`
- `useEffect(() => { ... }, [groupId])`:
  - Создать канал: `supabase.channel(\`group:${groupId}\`)`
  - Три подписки `.on('postgres_changes', ...)` на одном канале:
    - `event: 'INSERT'`, table: 'messages', filter: `group_id=eq.${groupId}` → `onInsert(payload.new as RealtimeMessage)`
    - `event: 'UPDATE'`, те же параметры → `onUpdate(payload.new as RealtimeMessage)`
    - `event: 'DELETE'` → `onDelete((payload.old as { id: string }).id)`
  - `.subscribe()`
  - `channelRef.current = channel`
  - Cleanup: `return () => { supabase.removeChannel(channel) }`
- Функция обязана подписываться ровно ОДИН РАЗ на группу (правило CLAUDE.md). groupId в deps.

**src/hooks/useTyping.ts**

Импорты: `useEffect, useRef, useState, useCallback` из react; `createClient` из `@/lib/supabase/client`.

Интерфейс:
```typescript
interface TypingUser {
  userId: string
  displayName: string
}
```

Функция `useTyping(groupId: string, currentUserId: string, currentUserName: string)`:

Возвращает `{ typingUsers: TypingUser[], setTyping: (typing: boolean) => Promise<void> }`.

Внутри:
- `const supabase = createClient()`
- `const channelRef = useRef<ReturnType<typeof supabase.channel> | null>(null)`
- `const [typingUsers, setTypingUsers] = useState<TypingUser[]>([])`

`useEffect(() => { ... }, [groupId, currentUserId])`:
- `const channel = supabase.channel(\`typing:${groupId}\`, { config: { presence: { key: currentUserId } } })`
- Подписка на presence sync:
  ```typescript
  channel.on('presence', { event: 'sync' }, () => {
    const state = channel.presenceState<{ typing: boolean; displayName: string }>()
    const typing = Object.entries(state)
      .filter(([uid, presences]) => uid !== currentUserId && presences[0]?.typing)
      .map(([uid, presences]) => ({ userId: uid, displayName: presences[0].displayName }))
    setTypingUsers(typing)
  })
  ```
- `channel.subscribe()`
- `channelRef.current = channel`
- Cleanup:
  ```typescript
  return () => {
    channelRef.current?.untrack()
    supabase.removeChannel(channel)
  }
  ```
  Важно: `untrack()` ПЕРЕД `removeChannel` — иначе typing остаётся в presence state у других (Pitfall 5).

`const setTyping = useCallback(async (typing: boolean) => { ... }, [currentUserName])`:
- Если `!channelRef.current` — return
- `await channelRef.current.track({ typing, displayName: currentUserName })`

Debounce для вызова `setTyping(false)` реализуется в MessageInput (не здесь) — хук только предоставляет функцию.

Запрещено: использовать `any`, создавать второй Presence-канал на ту же группу.
