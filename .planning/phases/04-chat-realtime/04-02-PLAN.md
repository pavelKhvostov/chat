Realtime:
```typescript
useRealtime({
  groupId,
  onInsert: (msg) => {
    setMessages((prev) => {
      // Дедупликация: не добавлять если уже есть (Pitfall 1)
      if (prev.some((m) => m.id === msg.id)) return prev
      // Realtime возвращает только базовые поля без JOIN
      // Поэтому добавляем с пустыми relations — они подтянутся при следующем fetchMessages
      // или можно отдельно fetch одного сообщения. Для v1: добавить с пустыми.
      const enriched = {
        ...msg,
        sender: { id: msg.sender_id, display_name: 'Загрузка...', avatar_url: null },
        reply: null,
        reactions: [],
        reads: [],
      } as MessageWithRelations
      return [...prev, enriched]
    })
  },
  onUpdate: (msg) => {
    setMessages((prev) =>
      prev.map((m) => m.id === msg.id ? { ...m, ...msg } : m)
    )
  },
  onDelete: (id) => {
    // Soft delete — update local state с deleted_at
    setMessages((prev) =>
      prev.map((m) => m.id === id ? { ...m, deleted_at: new Date().toISOString() } : m)
    )
  },
})
```

ВАЖНО по onInsert: Realtime payload.new содержит только колонки таблицы messages (без JOIN). Это нормально для v1 — сообщение появляется, но имя отправителя будет временным. При следующем обновлении страницы или reload данные будут корректными. Для продакшена — загружать одно сообщение через отдельный fetchMessage(id), но для v1 допустимо.
